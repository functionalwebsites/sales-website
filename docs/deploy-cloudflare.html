Deploy to Cloudflare Pages

Host your site on Cloudflare Pages for lightning-fast global performance, automatic HTTPS, and zero cold starts.

What is Cloudflare Pages?

Cloudflare Pages is a JAMstack hosting service that serves your static site from Cloudflare's global edge network. Your site loads fast from anywhere in the world.

Benefits:

  • ⚡ Global CDN (incredibly fast)
  • 🔒 HTTPS by default
  • 💰 Free tier with generous limits
  • 🚀 Automatic deployments from GitHub
  • 🌐 Custom domains included
  • 🔧 Serverless functions (optional)

Prerequisites

  • A Cloudflare account (free at dash.cloudflare.com)
  • Your site files (exported from Site Builder)
  • (Optional) GitHub repository with your site

Deployment Methods

Option 1: Direct Upload (Fastest)

Step 1: Export Your Site
In the Site Builder, click File > Export. Download the ZIP file.
Step 2: Login to Cloudflare
Go to dash.cloudflare.com and sign in (or create a free account).
Step 3: Go to Pages
Click "Pages" in the left sidebar, then "Create a project".
Step 4: Upload Your Site
  1. Select "Upload assets"
  2. Drag and drop your extracted site files (or click to select)
  3. Make sure index.html is included
  4. Click "Deploy site"
✅ Done! Your site is live at project-name.pages.dev. It may take a minute to build.

Option 2: GitHub Integration (Recommended for Updates)

This method automatically redeploys when you push to GitHub.

Step 1: Create a GitHub Repository
Follow the GitHub Pages guide to create a repository with your site files.
Step 2: Connect GitHub to Cloudflare
  1. Go to Cloudflare Dashboard
  2. Click "Pages" > "Create a project"
  3. Select "Connect to Git"
  4. Authorize GitHub when prompted
Step 3: Select Your Repository
  1. Choose your GitHub account
  2. Select your site repository
  3. Cloudflare auto-detects your settings
  4. Click "Save and deploy"

That's it! Every time you push to GitHub, your site automatically redeploys.

Using Your Custom Domain

Connect a Domain to Cloudflare

  1. Go to Cloudflare Dashboard > Pages > Your Project
  2. Click "Custom domains"
  3. Click "Add domain" and enter your domain (e.g., mysite.com)
  4. Cloudflare provides nameserver records
  5. Go to your domain registrar and update nameservers to point to Cloudflare
  6. Wait for propagation (5-48 hours)

Your site now loads at your custom domain with free HTTPS!

Using an Existing Cloudflare Domain

If your domain is already on Cloudflare, just add it to your Pages project—instant activation!

Custom Build Settings

For advanced deployments, configure build settings:

  1. Go to Pages Project > Settings > Build & deployment
  2. Build command: Leave blank (for static HTML)
  3. Build output directory: Leave blank or set to /
  4. Save

For static HTML sites, no build process is needed.

Environment Variables & Secrets

If your site uses custom code or APIs:

  1. Go to Settings > Environment variables
  2. Add variables (e.g., API keys)
  3. They're available during builds and in edge functions

Automatic Deployments

When using GitHub integration:

  • Push to your main branch → Site redeploys automatically
  • Create a pull request → Cloudflare builds a preview
  • Merge PR → Live site updates

Deploy multiple times per day with zero downtime!

Analytics & Monitoring

Cloudflare Pages includes free analytics:

  • Requests served
  • Data transferred
  • Performance metrics

View in Analytics tab of your project.

Troubleshooting

Site Not Showing?

  • Verify index.html is at the root
  • Check "Pages" for build errors
  • Try clearing browser cache

Broken Links or Assets?

Use relative paths in your HTML (e.g., ./images/photo.jpg instead of absolute paths).

Domain Not Connecting?

  • Wait for DNS propagation (48 hours max)
  • Verify nameservers are correct in your registrar
  • Check Cloudflare domain settings

Best Practices

  • Use GitHub integration for automatic deployments
  • Test locally before pushing
  • Use meaningful commit messages
  • Monitor your site's analytics
  • Enable Cloudflare security features (optional)
💡 Pro Tip: Cloudflare Pages includes Workers (serverless functions). Learn more at developers.cloudflare.com/workers.
