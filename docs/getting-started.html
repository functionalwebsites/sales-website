Getting Started

Learn the fundamentals of building a static website with the Functional Websites Site Builder.

Welcome to the Site Builder

The Functional Websites Site Builder is a powerful, open web tool that helps you create fast, secure static websites without code—or with code if you want it. This guide covers the basics to get you building in minutes.

Creating Your First Site

Step 1: Open the Builder
Visit /site-builder to start. No sign-up required for the free tier.
Step 2: Choose a Template
Select from built-in starter templates (Business, Portfolio, Blog, etc.) or start with a blank canvas. Templates give you a head start with common page layouts.
Step 3: Add Pages
Create pages like Home, About, Services, Contact, etc. Each page is independent and editable. Organize them in your site structure.
Step 4: Design with Blocks
Drag and drop blocks (sections, text, images, buttons, forms) onto your pages. Customize colors, fonts, spacing, and layout using the visual editor. No code required!
Step 5: Preview & Export
Preview your site locally, then export as HTML files or deploy directly to GitHub/Cloudflare.

Editor Interface

Toolbar (Top)

The toolbar contains:

  • File Operations: New, Open, Save, Export
  • Undo/Redo: Revert or redo changes
  • Preview: View your site in browser
  • Blocks Panel: Browse available blocks to add
  • Pages Panel: Create and manage pages

Canvas (Center)

The canvas is your design area. Click any element to select and edit it. Use handles to resize. Right-click for context menu options.

Properties Panel (Right)

When an element is selected, the properties panel shows editing options:

  • Text: Edit content, font, size, weight, color
  • Colors: Background, border, shadow
  • Spacing: Padding, margin
  • Layout: Width, alignment, flex options
  • Advanced: Custom CSS, ID, classes

Working with Blocks

What Are Blocks?

Blocks are reusable components—sections, cards, headers, forms, galleries, etc. They're the building blocks of your pages. The builder comes with standard blocks, and you can create or import custom ones.

Adding Blocks

Click the "Blocks" button in the toolbar to open the block library. Drag a block onto your canvas. Edit its content and styling in the properties panel.

Nesting Blocks

Blocks can contain other blocks. For example, a "Hero" section block might contain a heading block and a button block. This nesting creates complex, organized layouts.

Managing Pages

Create a Page

Click "Pages" in the toolbar, then "+ New Page". Give it a name (e.g., "About") and choose a template. The URL slug is auto-generated but can be customized.

Page Settings

Each page has settings for:

  • Title (for browser tab and SEO)
  • Meta description (for search results)
  • URL slug (the web address)
  • Template (which layout to use)

Page Navigation

Use the page tree to navigate between pages, reorder them, and manage site structure.

Saving & Exporting

Save Locally

Click "File > Save" to save your project locally as a .fw file. This allows you to reopen and edit later.

Export as HTML

Click "File > Export" to download your entire site as a ZIP file containing all HTML, CSS, and media. Extract and deploy anywhere.

Deploy Directly

Use the deployment guides below to publish your site to GitHub Pages or Cloudflare with one click.

💡 Pro Tip: Your site is 100% portable HTML. No lock-in. Whether you export, deploy to GitHub, Cloudflare, or anywhere else—it's yours to keep and control.

Key Concepts

Static HTML

Your site is pure HTML, CSS, and JavaScript (no database). This makes it:

  • ⚡ Fast: Instant page loads, no server processing
  • 🔒 Secure: No backend vulnerabilities
  • 💰 Cheap: Free or pennies to host
  • 📦 Portable: Deploy anywhere

Responsive Design

All templates and blocks are mobile-responsive by default. Your site looks great on phones, tablets, and desktops.

SEO Ready

Every page supports:

  • Meta titles and descriptions
  • Heading hierarchy (H1, H2, H3...)
  • Open Graph tags for social sharing
  • XML sitemap generation

Frequently Asked Questions

Do I need to know code?

No! The visual editor handles everything. But if you know HTML/CSS, you can add custom code for advanced customization.

Can I use my own domain?

Yes. When you deploy to GitHub Pages or Cloudflare, you can point your custom domain there. Both support custom domains easily.

Can I convert my site to something else later?

Absolutely. Export your site as HTML and migrate to any other platform. No lock-in.

What about dynamic content?

Static sites are perfect for content that doesn't change frequently. For dynamic content (databases, user accounts, real-time data), consider a backend solution or a headless CMS.
