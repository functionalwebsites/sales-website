Custom Templates

Create consistent page layouts and speed up your design workflow with reusable templates.

What Are Custom Templates?

Templates are pre-designed page layouts that define the structure, styling, and default content for pages. Instead of building each page from scratch, use a template and focus on content customization.

Examples of custom templates:

  • Blog Post Template (with hero, featured image, author bio)
  • Product Page Template (images, specs, pricing, reviews)
  • Service Page Template (overview, features, call-to-action)
  • Team Member Template (photo, bio, contact)
  • Testimonial Template (quote, author, company)

Built-in Templates

The site builder ships with starter templates:

  • Business: Perfect for service-based companies. Includes about, services, team, contact pages.
  • Portfolio: Showcase your work with project gallery, case studies, and bio.
  • Blog: Multi-page blog with home, article template, and archive.
  • Landing Page: High-conversion focused with hero, features, testimonials, CTA.
  • E-Commerce: Basic product listings, shopping cart, checkout (static version).
  • Blank: Start from scratch with minimal styling.

Using Templates

Create a New Page from a Template

  1. Click "Pages" in the toolbar
  2. Click "+ New Page"
  3. Choose a template from the gallery
  4. Customize the page layout and content

Apply a Template to an Existing Page

  1. Open the page you want to update
  2. Right-click on the page and select "Change Template"
  3. Choose a new template
  4. Your content is preserved where possible; new layout structure is applied

Creating Custom Templates

⭐ Pro Feature: Creating and uploading custom templates requires the Pro plan. Free users can use built-in templates.

Create a Template from a Page

  1. Design a page with the layout you want to reuse
  2. Right-click on the page and select "Save as Template"
  3. Name your template (e.g., "Blog Post")
  4. Add a description and preview thumbnail
  5. Choose which content should be editable vs. fixed
  6. Click "Save Template"

Define Editable Regions

When saving a template, specify which parts are editable:

  • Editable: Text, images, colors that users can change
  • Fixed: Layout, spacing, styling that stays consistent

This ensures consistency while allowing flexibility. For example, a blog template might have a fixed header but editable post content.

Your Template is Ready

Your template appears in the templates gallery. When you create a new page, you can select your custom template as the starting point.

Template Best Practices

Keep It Generic

Design templates for multiple use cases. A "Product Card" template is better than a "Nike Shoe Card" template.

Define Clear Regions

Mark sections as editable or fixed. Fixed regions provide consistency; editable regions allow customization.

Responsive Design

Ensure your template looks good on all screen sizes. Test on mobile before saving.

Minimal Default Content

Use placeholder text and images. Let users fill in their own content rather than having to delete yours.

Name Descriptively

Use clear names: "Blog Post", "Service Overview", "Team Member" etc. Avoid generic names like "Page Template".

Publishing Templates to Marketplace

Pro users can share custom templates with the community:

  1. Go to Account > My Templates
  2. Select your template and click "Publish to Marketplace"
  3. Add details:
    • Description of the template
    • Category (blog, portfolio, business, etc.)
    • Multiple preview images
    • Pricing (free or paid)
  4. Submit for review

Once approved, earn revenue from users who download your template!

Template Marketplace

Discover professional templates:

  • Visit /marketplace
  • Filter by category (blog, portfolio, business, etc.)
  • Preview with screenshots and live demos
  • Click "Get Template" to add to your project

Advanced: Template JSON Structure

Templates are stored as JSON with metadata:

  • name: Template title
  • description: What the template is for
  • category: Blog, Portfolio, Business, etc.
  • structure: Page layout and blocks
  • editableRegions: Which parts users can customize
  • preview: Thumbnail image

Advanced users can edit JSON directly. Find templates in /site-builder/templates/.

Workflow Example: Building a Blog

Step 1: Create a Blog Post Template

  • Design a page with: hero image, title, date, author, content, related posts
  • Save as template "Blog Post"
  • Mark title, content, author, date as editable

Step 2: Create Blog Posts

  • Create new pages using "Blog Post" template
  • Each post has consistent layout
  • Only customize content, not design

Step 3: Create Blog Home

  • Create a "Blog Home" page listing all posts
  • Use custom blocks for post cards

Result: A professional blog with consistent design in minutes!

FAQ

Can I update a template after creating pages from it?

Template updates don't automatically apply to existing pages. This is by design—each page is independent. If you need to update all pages, you'll need to manually edit them or recreate pages from the updated template.

Can I delete a template?

Yes, but it won't affect pages already created from that template. Pages are copies, not linked to the template.

Can I nest templates?

Not directly, but you can use custom blocks within templates for modularity.

What's the difference between templates and blocks?

Templates: Full page layouts, define the overall page structure. Blocks: Reusable components within pages. Use templates for page layout, blocks for modular content.

💡 Pro Tip: Combine templates and custom blocks for maximum efficiency. Use templates for page structure, blocks for repeating components.
