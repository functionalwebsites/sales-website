Export & Self-Host

Export your site as plain HTML and deploy anywhere—your domain, your server, your choice. 100% portable, 100% yours.

Why Export Your Site?

  • ✅ Ownership: Complete control of your files
  • ✅ Portability: Run on any server, anywhere
  • ✅ No Lock-in: Not dependent on our platform
  • ✅ Archival: Keep a permanent backup locally
  • ✅ Custom Hosting: Use your preferred hosting provider

Step 1: Export Your Site

  1. Open your project in the Site Builder
  2. Click File > Export (top toolbar)
  3. A ZIP file downloads containing all your site files
  4. Extract the ZIP to a folder on your computer

What's in the Export?

  • index.html, about.html, etc.: Your site pages
  • css/: All stylesheets
  • js/: JavaScript files
  • images/: Your images and media
  • .htaccess (optional): Server configuration

That's it—pure HTML, CSS, JavaScript. No dependencies. No build process.

💾 Backup Tip: Keep this ZIP file in a safe place (Google Drive, Dropbox, etc.) as a permanent backup of your site.

Step 2: Choose a Hosting Provider

Any static host works. Here are popular options:

Netlify (netlify.com)
Free tier available. Drag-drop upload. Automatic deployments. Great for static sites.
Vercel (vercel.com)
Built for performance. Simple deployments. Free plan included.
AWS S3 + CloudFront
Pay-as-you-go. Great for high traffic. More complex setup.
Firebase Hosting (firebase.google.com)
Google's hosting. Free tier. Easy setup for Firebase users.
DigitalOcean (digitalocean.com)
Traditional VPS. Full control. Starting at $5/month.
Bluehost, GoDaddy, Namecheap
Traditional web hosting. Often includes domain. Affordable.
Your Own Server
Complete freedom. Requires technical knowledge. Use NGINX or Apache.

Step 3: Upload Your Files

The process varies by provider, but generally:

Option A: Drag & Drop (Netlify, Vercel, etc.)

  1. Login to your hosting provider
  2. Find the upload/deploy button
  3. Drag your extracted folder onto it
  4. Done! Site goes live in seconds

Option B: Git/GitHub (Netlify, Vercel, etc.)

  1. Push your files to GitHub
  2. Connect your provider to GitHub
  3. Automatic deployments on push

Option C: FTP/SFTP (Traditional Hosting)

  1. Get FTP credentials from your host
  2. Use an FTP client (Filezilla, Cyberduck)
  3. Upload files to public_html/ or www/ folder
  4. Site accessible at your domain

Option D: Command Line / Deployment Scripts

Many hosts provide CLI tools or scripts. Refer to their documentation.

Step 4: Connect Your Domain

If You Already Own a Domain

  1. Get the DNS records from your hosting provider
  2. Login to your domain registrar (GoDaddy, Namecheap, etc.)
  3. Update DNS records to point to your host
  4. Wait for propagation (5-48 hours)

If You Don't Have a Domain

  1. Buy a domain from a registrar (namecheap.com, etc.)
  2. Configure DNS to point to your host
  3. Many hosts sell domains too—check if cheaper bundled

Your site is now live at your custom domain!

Post-Deployment Checklist

  • ☑️ Test all pages load correctly
  • ☑️ Check links work (no 404 errors)
  • ☑️ Verify images display
  • ☑️ Test on mobile devices
  • ☑️ Check forms work (if applicable)
  • ☑️ Enable HTTPS (most hosts do automatically)
  • ☑️ Monitor performance (GTmetrix, PageSpeed Insights)

Updating Your Site

After Making Changes in Site Builder

  1. Export again from Site Builder
  2. Upload new files to your host
  3. Done—site updates automatically

Automating Updates (Recommended)

Instead of manual exports, use Git:

  1. Export to a GitHub repository
  2. Connect your host to auto-deploy from GitHub
  3. Each push automatically deploys

Performance Optimization

Image Optimization

  • Compress images before uploading (TinyPNG, Squoosh)
  • Use modern formats (WebP)
  • Resize to appropriate dimensions

Caching

  • Enable browser caching (most hosts have settings)
  • Use a CDN if available

CSS/JS Minification

The Site Builder should already minify these, but double-check in export.

Security Best Practices

  • 🔒 Enable HTTPS (required for most hosts now)
  • 🔒 Remove any sensitive information from client-side code
  • 🔒 Keep backups in multiple locations
  • 🔒 Monitor for broken links (404 errors)

SEO for Self-Hosted Sites

Ensure search engines can index your site:

  • Create a sitemap.xml (Site Builder may include this)
  • Create a robots.txt file
  • Submit sitemap to Google Search Console
  • Verify all pages are indexable

Sample robots.txt

Create a robots.txt in your root folder:

User-agent: *
Allow: /
Sitemap: https://yoursite.com/sitemap.xml

Troubleshooting

Site Not Loading?

  • Verify index.html is in root directory
  • Check DNS propagation (dns-checker.com)
  • Verify HTTPS certificate is valid

Broken Links or Assets?

  • Check file paths are relative (not absolute)
  • Verify all files were uploaded
  • Check file permissions on server

Slow Performance?

  • Enable gzip compression
  • Use a CDN
  • Optimize images
  • Use browser caching
