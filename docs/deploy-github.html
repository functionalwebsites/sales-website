Deploy to GitHub Pages

Host your site for free on GitHub Pages with automatic deployments from your GitHub repository. Perfect for portfolios, blogs, and open-source projects.

What is GitHub Pages?

GitHub Pages is a free static site hosting service. Push your HTML files to a GitHub repository, and they're automatically published at username.github.io/repo-name or your custom domain.

Benefits:

  • ✅ Free hosting
  • ✅ Automatic deployments from Git
  • ✅ Custom domain support
  • ✅ HTTPS by default
  • ✅ Version control built-in

Prerequisites

  • A GitHub account (free at github.com)
  • Git installed on your computer
  • Your site exported from the Site Builder as HTML files

Step-by-Step Setup

Step 1: Export Your Site
In the Site Builder, click File > Export. Download the ZIP file containing all your site files.
Step 2: Create a GitHub Repository
Go to github.com/new and create a new repository. Name it something descriptive (e.g., my-portfolio-site). Make it public. Don't add a README yet.
Step 3: Prepare Your Files
  1. Extract the ZIP file from Step 1
  2. Copy all files and folders into your repository folder on your computer
  3. Make sure index.html is at the root level
Step 4: Initialize Git & Push
Open terminal in your repository folder and run:
git init git add . git commit -m "Initial site commit" git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git git branch -M main git push -u origin main
Step 5: Enable GitHub Pages
  1. Go to your repository on GitHub
  2. Click Settings (top right)
  3. Scroll to "Pages" (left sidebar)
  4. Under "Source", select "Deploy from a branch"
  5. Select main branch and /root folder
  6. Click Save
🎉 Done! Your site is now live at https://username.github.io/repo-name. It may take a few minutes to build.

Using Your Custom Domain

Connect Your Domain

  1. Own a domain (e.g., from GoDaddy, Namecheap, etc.)
  2. Go to your domain registrar's DNS settings
  3. Create DNS records pointing to GitHub:
    A records: 185.199.108.153 185.199.109.153 185.199.110.153 185.199.111.153
  4. Wait for DNS to propagate (5-48 hours)

Configure in GitHub

  1. Go to Settings > Pages
  2. Under "Custom domain", enter your domain (e.g., mysite.com)
  3. GitHub automatically enables HTTPS

Your site now loads at your custom domain!

Automatic Deployments

Every time you push changes to GitHub, your site automatically redeploys:

git add . git commit -m "Update site content" git push

Your changes go live within seconds!

Updating Your Site

Option A: Use Git (Recommended)

  1. Export updated files from Site Builder
  2. Replace files in your repo folder
  3. Git add, commit, push

Option B: Edit Directly on GitHub

  1. Go to your repository
  2. Click file to edit
  3. Click pencil icon to edit
  4. Commit changes (auto-deploys)

Troubleshooting

Site Not Showing?

  • Check that index.html is in the root folder
  • Wait a few minutes for the build to complete
  • Go to Settings > Pages to check build status

Broken Links After Deployment?

If your site is at username.github.io/repo-name/, make sure internal links include the repo path or use relative paths (e.g., ./about.html).

HTTPS Not Working?

GitHub Pages automatically provides HTTPS. If it's not working, go to Settings > Pages and ensure "Enforce HTTPS" is checked.

Best Practices

  • Keep a backup of your site locally
  • Use Git to track all changes
  • Create a .gitignore file to exclude unnecessary files
  • Use meaningful commit messages
  • Test locally before pushing
💡 Pro Tip: You can enable branch protection in GitHub Settings to require reviews before pushing to main.
