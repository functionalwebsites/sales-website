What is GitHub Pages?

GitHub Pages is free static site hosting provided by GitHub. Push your HTML files to a repository, and they're automatically published at username.github.io/repo or your custom domain.

Prerequisites

  • GitHub account (free at github.com)
  • Git installed on your computer
  • Your site exported from the Site Builder as HTML

Step 1: Export Your Site

  1. Open your project in the Site Builder
  2. Click File > Export
  3. Download the ZIP file
  4. Extract it to a folder on your computer

Step 2: Create a GitHub Repository

  1. Go to github.com/new
  2. Name your repository (e.g., my-portfolio-site)
  3. Set it to Public
  4. Click Create repository

Step 3: Initialize Git & Push Your Files

Open Terminal, navigate to your site folder, and run:

git init
git add .
git commit -m "Initial site commit"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git
git push -u origin main

Replace: YOUR-USERNAME with your GitHub username and YOUR-REPO-NAME with your repository name.

Step 4: Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click Settings tab
  3. Scroll down to Pages section
  4. Under "Branch", select main
  5. Click Save
  6. Your site is now live at https://username.github.io/repo-name

Step 5: Connect Your Custom Domain (Optional)

  1. Buy a domain from a registrar (Namecheap, GoDaddy, etc.)
  2. Go to your domain's DNS settings
  3. Create a CNAME record pointing to username.github.io
  4. In your repo Settings > Pages, enter your custom domain
  5. Wait a few hours for DNS to propagate
  6. Your site is now accessible at your custom domain

Updating Your Site

After making changes in the Site Builder:

  1. Export your updated site
  2. Replace files in your local folder
  3. Run these Git commands:
git add .
git commit -m "Update site"
git push origin main

Your site automatically deploys within seconds.

Troubleshooting

Site Not Showing?

  • Verify index.html is in the root of your repository
  • Check that Pages is enabled in Settings
  • Refresh the page after a minute or two

Broken Links or Images?

  • Make sure all HTML, CSS, images, and JS files were uploaded
  • Check file paths are relative (not absolute)
  • Use lowercase filenames (GitHub is case-sensitive on some systems)

HTTPS Not Working?

  • GitHub Pages always provides free HTTPS
  • If you have a custom domain, wait up to 24 hours for the certificate
  • Clear your browser cache and try again

Best Practices

  • Commit Often: Make small commits with clear messages
  • Use .gitignore: Exclude large files or system files
  • Test Locally: Use a local server to test before pushing
  • Monitor Performance: Use PageSpeed Insights to check speed
